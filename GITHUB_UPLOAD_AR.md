# رفع المشروع لـ GitHub والحصول على APK

هذه الحزمة **جاهزة للرفع مباشرة**. التطبيق يتصل بالـ Space مباشرة
(`abdo96-chatterbox.hf.space`) — لا حاجة لسيرفر وسيط.

## ✅ ما بداخل الحزمة (كله محدّث ومؤكد)
- `lib/services/api_service.dart` — يتصل بالـ Space عبر Gradio API
- `lib/main.dart` — رابط الـ Space مثبّت
- `.github/workflows/build.yml` — يبني APK تلقائياً
- `pubspec.yaml` — إصدار record 6 (بلا أخطاء بناء)
- كل الشاشات والمنطق

> `api-server.js` موجود لكن **غير مستخدم** — التطبيق يتصل بالـ Space مباشرة.
> تجاهله (أو احذفه).

---

## الطريقة الأولى: عبر واجهة GitHub (بدون أوامر)

1. افتح مستودعك على github.com
2. احذف الملفات القديمة: كل شيء داخل `lib/` والملفات القديمة
   (أو أنشئ مستودعاً جديداً فارغاً)
3. اسحب وأفلت محتويات مجلد `jisr` المفكوك (بما فيه مجلد `lib`
   و `.github` و `pubspec.yaml`)
4. اكتب رسالة commit مثل "تحديث كامل" واضغط **Commit changes**

## الطريقة الثانية: عبر الطرفية (أضمن)

```bash
# 1) ادخل مجلد مشروعك المحلي المربوط بـ GitHub
cd مسار-مشروعك

# 2) انسخ كل ملفات jisr المفكوكة فوق مشروعك
#    (استبدل القديمة بالجديدة)

# 3) ارفع
git add -A
git commit -m "تحديث كامل: اتصال مباشر بالـ Space + إصلاحات البناء"
git push
```

---

## بعد الرفع — احصل على APK

1. افتح تبويب **Actions** في مستودعك
2. انتظر آخر تشغيل يكتمل بعلامة ✅ خضراء (~5-8 دقائق)
3. اضغط على التشغيل → انزل لأسفل → قسم **Artifacts**
4. حمّل **jisr-release-apk** (ملف مضغوط بداخله الـ APK)
5. فك الضغط، وثبّت الـ APK على الهاتف

> ⚠️ **مهم:** احذف التطبيق القديم من الهاتف **تماماً** قبل تثبيت الجديد.

---

## للتأكد أن الرفع نجح

افتح على GitHub في المتصفح: `lib/services/api_service.dart`
وابحث (Ctrl+F) عن كلمة **`gradio_api`**.
- وجدتها → الرفع نجح ✅، انتظر البناء
- لم تجدها → الملفات القديمة ما زالت، أعد الرفع
